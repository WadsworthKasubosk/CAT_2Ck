## 一、项目整体架构

### 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | Vue 2 + ElementUI + ECharts + axios | 单页面应用，所有业务逻辑集中在 Content.vue |

## 二、推理调用链路

```
[用户浏览器]
    │  POST /upload (FormData: dcm文件 + patient_id)
    ▼
[app.py]
    │  保存文件到 uploads/ 和 tmp/ct/
    │  调用 core.main.c_main(path, model)
    ▼
[core/main.py  c_main()]
    ├── process.pre_process(path)      # 读取dcm → 裁剪ROI[270:430,200:300] → 归一化 → 保存png到tmp/image/
    ├── predict.predict(data, model)   # U-Net推理 → 二值化(阈值0.5) → 保存mask到tmp/mask/
    ├── process.last_process(name)     # 原图+mask → cv2.findContours/drawContours → 保存到tmp/draw/
    └── get_feature.main(name)         # 读取dcm+mask → 计算24项特征 → 返回dict
    │
    │  返回: {status, image_url, draw_url, image_info}
    ▼
[app.py]
    │  若前端传了 patient_id → 写入 diagnosis_records 表
    │  返回 JSON（含 record_id）
    ▼
[用户浏览器]
    ├── 显示原CT图 (image_url)
    ├── 显示标注图 (draw_url)
    ├── 显示特征值表格 (image_info 24项)
    └── 显示面积/周长趋势图（历史数据从数据库查询）
```

## 三、数据库设计

### 表 1：`patients` — 患者信息

```sql
CREATE TABLE IF NOT EXISTS patients (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    name       TEXT    NOT NULL,
    gender     TEXT    NOT NULL,
    age        INTEGER,
    phone      TEXT,
    body_part  TEXT    DEFAULT '直肠',
    created_at TEXT    DEFAULT CURRENT_TIMESTAMP
);
```

### 表 2：`diagnosis_records` — 诊断记录 + 特征值

```sql
CREATE TABLE IF NOT EXISTS diagnosis_records (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    patient_id   INTEGER NOT NULL,
    dcm_filename TEXT,
    image_url    TEXT,
    draw_url     TEXT,
    area         REAL,         -- 单独建列，便于趋势查询
    perimeter    REAL,         -- 单独建列，便于趋势查询
    image_info   TEXT,         -- 完整 24 项特征 JSON
    doctor_note  TEXT,
    created_at   TEXT    DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (patient_id) REFERENCES patients(id)
);
```

### 表 3：`llm_reports` — LLM 辅助诊断报告

```sql
CREATE TABLE IF NOT EXISTS llm_reports (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    diagnosis_id  INTEGER NOT NULL,
    prompt        TEXT,
    advice        TEXT,
    model_name    TEXT,
    created_at    TEXT    DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (diagnosis_id) REFERENCES diagnosis_records(id)
);
```

### 表关系

```
patients (1) ──────< (N) diagnosis_records (1) ──────< (N) llm_reports
```

### 趋势分析 SQL（不单独建表，从 diagnosis_records 派生）

```sql
SELECT area, perimeter, created_at
FROM diagnosis_records
WHERE patient_id = ?
ORDER BY created_at ASC;
```

## 五、核心接口详细说明

### POST /upload — 上传推理（扩展后）

**请求**：`multipart/form-data`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `file` | File | 是 | .dcm 格式 CT 文件 |
| `patient_id` | int | 否 | 若传入则推理结果写入数据库 |

**响应（不传 patient_id，向后兼容）**：
```json
{
    "status": 1,
    "image_url": "http://127.0.0.1:5003/tmp/image/test001.png",
    "draw_url": "http://127.0.0.1:5003/tmp/draw/test001.png",
    "image_info": { "area": ["面积", 1086.0], "perimeter": ["周长", 159.32], "...": "24项特征" }
}
```

**响应（传 patient_id，多返回 record_id）**：
```json
{
    "status": 1,
    "image_url": "...",
    "draw_url": "...",
    "image_info": { ... },
    "record_id": 1
}
```

### POST /api/patients — 新增患者

**请求体**：
```json
{ "name": "李明", "gender": "男", "age": 29, "phone": "13220986785", "body_part": "直肠" }
```

**响应**：
```json
{ "status": 1, "data": { "id": 1, "name": "李明", "gender": "男", "age": 29, "phone": "13220986785", "body_part": "直肠", "created_at": "2026-03-13 02:22:33" } }
```

### GET /api/patients/<id>/trend — 趋势数据

**响应**：
```json
{
    "status": 1,
    "data": {
        "x_axis": ["2026-01-01", "2026-02-01", "2026-03-01"],
        "area": [1300.0, 1092.0, 980.5],
        "perimeter": [250.0, 159.32, 135.2],
        "trend": { "area": "下降", "perimeter": "下降" }
    }
}
```

### POST /api/diagnosis/<id>/llm-advice — LLM 辅助建议

**无需请求体**，接口自动从数据库获取特征值和历史趋势构建 prompt。

**响应**：
```json
{
    "status": 1,
    "data": {
        "report_id": 1,
        "advice": "根据本次CT检查结果分析：\n\n1. **当前肿瘤特征分析** ...\n2. **与历史诊断对比** ...\n3. **建议** ...",
        "model_name": "qwen-plus",
        "disclaimer": "⚠️ 以上内容由 AI 生成，仅供辅助参考，不能替代专业医生的诊断和治疗方案。请遵医嘱。"
    }
}
```

```bash

pip install dashscope

# 方式 2：不安装，自动回退到 HTTP 请求（项目已有 requests）
```

### Prompt 结构

```
你是一名医学影像分析助手，请根据以下直肠肿瘤 CT 图像的分析结果，给出辅助诊断建议。

## 本次诊断的肿瘤区域特征值
- 面积: 1086.0
- 周长: 159.32
- 灰度均值: 128.5
- 似圆度: 0.85
- ...（全部 24 项特征）

该患者已有 N 次诊断记录：
- 肿瘤面积变化趋势：下降，历史值：[1300.0, 1092.0, 980.5]
- 肿瘤周长变化趋势：下降，历史值：[250.0, 159.32, 135.2]

## 请给出以下内容
1. 对当前肿瘤特征的简要分析
2. 与前几次诊断对比的变化情况
3. 建议的后续检查或关注事项
4. 需要提醒医生注意的异常指标（如有）
```
